Panel de operaciones (real)

Puerto correcto: en Docker Compose el front suele publicarse en :8280 y el gateway en :3000. http://localhost/ops-panel/ usa el puerto 80; si no mapeaste el 80 en .env, verás connection refused. Usá por ejemplo http://localhost:8280/ops-panel/ o http://localhost:3000/ops-panel/ (según FRONTEND_HOST_PORT / GATEWAY_HOST_PORT).

Health y circuit breakers se leen por HTTP al gateway interno. Los logs son salida real de docker logs. Apagar una base o un microservicio ejecuta la API de Docker sobre el socket montado (equivalente a docker stop). Definí OPS_PANEL_TOKEN en tu .env y pegalo abajo.

      Cómo probar circuit breakers (abierto / cerrado / semiabierto) y ver el estado
      

        
Estado en vivo (recomendado): en esta misma página, sección Monitoreo → Refrescar. En Tarjetas o Tabla breakers ves si cada breaker está Cerrado, Abierto o Semiabierto (viene del gateway: /api/health/breakers/<servicio>).

        
Desde terminal (puerto según tu .env, ejemplo gateway 3000):

        
curl -s http://localhost:3000/api/health/breakers/usuarios | jq

        
Prueba típica (laboratorio) para ver el circuito abierto en logs y en la tabla:

        
    
          
  1. Levantá el stack con todo running.
    2. 
          
  2. En Contenedores, hacé Stop en db-usuarios (o la base que use el servicio que mirás).
    3. 
          
  3. Generá varias peticiones que usen la base (por ejemplo login o GET /api/health en usuarios) hasta que fallen; el breaker usuarios-mysql puede pasar a Abierto tras varios fallos seguidos (umbral en el servicio, p. ej. CB_* en .env).
    4. 
          
  4. Acá, target usuarios-service → Traer logs → vista Solo lo importante: vas a ver líneas claras de circuit breaker y errores de red.
    5. 
          
  5. Start en db-usuarios; esperá unos segundos (p. ej. resetTimeout del breaker, ~15 s por defecto en usuarios). Refrescá monitoreo: suele aparecer Semiabierto y luego Cerrado si la base responde.
